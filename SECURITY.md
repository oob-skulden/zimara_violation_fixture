# Security Policy

## Important Notice

This repository **intentionally contains insecure code and configuration
patterns** for the purpose of testing and validating security detection tools.

These files are **NOT vulnerabilities** and **NOT accidental leaks**.

They exist solely as **test fixtures**.

---

## What This Repository Is

- A deliberate collection of insecure examples
- A regression and validation dataset for Zimara
- A safe environment for triggering security scanners

All credentials, secrets, and identifiers are:
- Synthetic
- Non-functional
- Created for detection testing only

---

## What This Repository Is NOT

- A production codebase
- A live system
- A source of real credentials or sensitive data

Please do **not** open security advisories or vulnerability reports for:
- Hardcoded secrets
- Weak configurations
- Unsafe practices intentionally present here

These findings are **expected by design**.

---

## Reporting Actual Issues

If you believe there is:
- A real, functioning credential
- Personally identifiable information
- Content unrelated to testing intent

Please open a standard GitHub issue with details.

Do **not** request CVE assignment for this repository.
